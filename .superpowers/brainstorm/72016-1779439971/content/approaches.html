Three RAG Architecture Approaches

Each solves the chunking + retrieval problem differently. Which fits a learning project best?

A

Better Baseline RAG

What changes from current: Paragraph-aware chunking (respects sentence boundaries) + dense embeddings + RAGAS evaluation added.

Pipeline: Question → Paragraph chunks → Dense retrieval (top-5) → GPT-4o answer → RAGAS score

Pros

  • Simple to build and understand
  • Good learning baseline to measure from
  • Fast to implement

Cons

  • Still misses context for hard questions
  • No keyword search — misses exact terms like "$97.69 billion"
  • Too simple to expose interesting RAG problems
B

Hierarchical + Hybrid RAG ⭐ Recommended

What changes from current: Two-level chunking (small for retrieval, large for context) + hybrid search (embeddings + BM25 keyword) + reranker + RAGAS evaluation.

Pipeline: Question → Hybrid search (dense + BM25) → top-20 candidates → Reranker → top-5 → Parent chunk lookup → GPT-4o answer → RAGAS score

Pros

  • Solves both problems you identified
  • Each component teaches one RAG concept
  • Modern production-grade pattern
  • RAGAS shows impact of each improvement

Cons

  • More components to understand
  • Harder to debug when something goes wrong
C

Agentic RAG

What changes from current: LLM decides what to retrieve and when. Multi-step retrieval — it can ask follow-up questions to itself before answering.

Pipeline: Question → LLM decides retrieval strategy → retrieve → LLM checks if context is enough → retrieve more if needed → answer

Pros

  • Handles complex multi-hop questions naturally
  • Most powerful for hard reasoning

Cons

  • Non-deterministic — hard to measure what improved
  • Very complex to debug
  • Overkill before you understand the basics

Why B is recommended for learning

Option B is the only one where you can turn each component on/off and measure the difference. Add BM25 → measure. Add reranker → measure. Each step teaches you exactly what that component contributes. That's the learning loop we designed from the start.
