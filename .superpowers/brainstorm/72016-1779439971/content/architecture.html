🔍 QUERY PHASE — runs every time a question is asked

❓
Question
"What was Tesla's revenue in 2024?"
→
🔢 Dense Search
Embed question → search ChromaDB by vector similarity → top 20 chunks
Good for: meaning & paraphrase
🔑 Sparse Search (BM25)
Match keywords "Tesla", "revenue", "2024" → top 20 chunks
Good for: exact terms & numbers
→
🔀 Merge Results
Combine both lists (up to 40 candidates)
using Reciprocal Rank Fusion
Why: covers both failure modes
⚖️ Reranker
Cross-encoder reads question + each candidate chunk together and scores true relevance.

40 in → picks best 5

Why not just use top-5 from search? Search finds candidates by similarity. Reranker reads them carefully and re-orders. Much more accurate.
→
📂 Parent Lookup
Each of the top-5 child chunks has a parent_id.

Retrieve the full parent section (larger context).

This is the "retrieve small, pass large" trick. Precision from child, context from parent.
→
🤖 GPT-4o
Receives:
• Original question
• 5 parent chunks

Generates a grounded answer using only the retrieved context.
→
📊
RAGAS
Scores the answer:
• Was the right chunk found? (retrieval)
• Did LLM stick to chunks? (faithfulness)
• Is the answer correct? (correctness)
